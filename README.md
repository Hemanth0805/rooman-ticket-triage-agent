# Support Ticket Triage Agent

This agent takes support ticket subject/body input and produces a ranked routing decision with category, urgency, team, confidence, and reasoning.

## What it does

- Reads support tickets from JSON.
- Classifies each ticket by category and urgency.
- Assigns a routing team.
- Writes structured output as CSV.

## Setup

1. Create and activate a Python virtual environment.

```bash
python -m venv venv
# Windows PowerShell
venv\Scripts\Activate.ps1
# macOS / Linux
source venv/bin/activate
```

2. Install dependencies.

```bash
pip install -r requirements.txt
```

3. (Optional) copy the `.env` example if you want to store your key locally.

```bash
copy .env.example .env
```

## Run the agent

### Mock mode (recommended for offline demo)

```bash
python agent.py --input data/tickets/sample_tickets.json --output outputs/routing_test.csv --mock
```

This produces sample output without an OpenAI key.

### Real mode with OpenAI

If you have an OpenAI API key, run:

```bash
python agent.py --input data/tickets/sample_tickets.json --output outputs/routing.csv --api-key "sk-your-valid-key"
```

If you set `OPENAI_API_KEY` in your environment or `.env`, you can omit `--api-key`.

## Output

- `outputs/routing.csv`: default real-mode output file
- `outputs/routing_test.csv`: sample output generated by mock mode

## Deliverables

- `agent.py`: CLI agent script
- `data/tickets/sample_tickets.json`: sample tickets
- `outputs/routing_test.csv`: sample routed output
- `README.md`: setup and usage instructions
- `.env.example`: API key configuration example

## Design notes and tradeoffs

- The agent is built as a small CLI glue layer around OpenAI.
- It supports both real API calls and offline mock mode for easy demoing.
- Real-mode depends on a valid OpenAI key and the current OpenAI Python client (`openai>=1.0.0`).
- With more time, I would add automated tests, richer ticket schema support, and a light rule-based fallback that combines model predictions with deterministic routing.
